# 🏺 MemoHack Skill: retroactive-distill
> 历史对话知识考古 · 坐标型入库 · 比对更新

---

## 🎯 核心定位

**本技能的唯一目标**：从当前 active 对话的 Context Window 中，提取对**下一个 AI 执行任务有直接价值的技术坐标**，与现有 `manual.md` 比对后，执行精准的新增或更新操作。

**不做的事**：
- ❌ 不写对话过程描述
- ❌ 不记录"我们发现了..."式的感悟
- ❌ 不重复已在 manual.md 中存在的内容

---

## 🔑 入库资格判定（过滤器）

从 Context Window 扫描出的每一条信息，必须通过以下全部判断才能入库：

```
问题1：下一个 AI 能否凭借这条记录直接执行操作？
        → 是 ✅ / 否 ❌（否则放弃）

问题2：这条记录是否包含具体的路径/命令/字段名/数值？
        → 是 ✅ / 否 ❌（否则放弃）

问题3：manual.md 中是否已有相同或更新的记录？
        → 没有 ✅（新增）/ 有旧版 ⚠️（更新）/ 已是最新 ❌（跳过）
```

---

## 🛠️ 执行流程

### Step 0：读取现有 manual.md

```bash
cat manual.md
```

将其内容作为"比对基准"，后续所有提炼结果都要与之比对。

---

### Step 1：从 Context 提炼候选坐标

**只关注以下 4 类高价值内容**：

#### A. 验证过的技术坐标
经由 Terminal 命令实际确认过的路径、命令、字段、连接信息：
```
格式：[简短标题] [#MH-TAG]
命令/路径：<具体内容>
验证结果：<关键输出数值或状态>
适用场景：<什么时候用这条>
```

#### B. 已修复的 Bug 模式
"报错类型 → 根本原因 → 一行修复"三元组：
```
格式：Bug修复 [#MH-DEBUG]
现象：<错误信息关键词>
原因：<根本原因>
修复：<具体命令或改动>
```

#### C. 禁止路径（Redlines）
经过验证、证明走不通的方向：
```
格式：⛔ 禁止 [#MH-DEBUG]
操作：<具体禁止的操作>
原因：<一句话说明为什么>
```

#### D. 架构决策
影响后续开发的重要约定或结构性发现：
```
格式：架构决策 [#MH-ARCH]
决策：<具体内容>
背景：<什么场景下适用>
```

---

### Step 2：比对 manual.md，产出差异清单

逐条对比提炼结果与 manual.md 现有内容：

| 比对结果 | 处理方式 |
|---|---|
| manual.md **没有** 此内容 | 标记为 `[NEW]`，准备新增 |
| manual.md **有旧版**，新信息更准确 | 标记为 `[UPDATE]`，准备覆盖 |
| manual.md **已是最新** | 标记为 `[SKIP]`，丢弃 |

---

### Step 3：生成补丁预览 Artifact

输出格式严格遵循：

```markdown
## 🏺 retroactive-distill 补丁预览

### [NEW] 标题 #MH-TAG
<坐标型内容>

### [UPDATE] 标题 #MH-TAG
原内容：<旧版摘要>
更新为：<新版内容>

---
共计：X 条新增，Y 条更新，Z 条跳过
等待用户回复 'Confirm' 后执行写入。
```

> [!IMPORTANT]
> 必须展示补丁预览并等待 'Confirm'，禁止直接写入。

---

### Step 4：写入 manual.md

用户确认后，调用 vibe_sync 执行入库：

```bash
# 每条记录单独调用，按类型分类
python3 skills/memohack/scripts/vibe_sync.py \
  --type MANUAL \
  --content "[坐标型内容，单条]" \
  --confirm
```

**[NEW] 条目**：直接写入对应 Section。
**[UPDATE] 条目**：先用 grep 定位旧条目行号，再用 vibe_sync 覆盖。

---

## 🚀 触发方式

在任意 active 对话框（当前或历史）中发送：

```
/distill
```

或自然语言：
```
请对本对话进行考古蒸馏，只提取对项目有用的坐标型内容写入 manual.md
```

---

## 📏 入库质量标准（最终检验）

写入 manual.md 前，逐条自问：

> **"下一个 AI 打开项目，读到这条记录，能否在 10 秒内知道该如何操作？"**

- ✅ 能 → 写入
- ❌ 不能 → 继续精炼或丢弃
